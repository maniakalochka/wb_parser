# WB Parser 

Парсер Wildberries, который по поисковому запросу собирает товары из каталога, догружает данные карточки и выгружает результат в Excel.

**Реализовано как CLI-утилита, без Docker!**

Проект рассчитан на сбор полей вроде:
- ссылка на товар
- артикул (nm_id)
- название
- цена 
- размеры
- общий остаток (stock_total)
- рейтинг и кол-во отзывов
- продавец (seller + ссылка)
- описание (из `card.json`)
- характеристики (JSON)
- страна производства (если есть)
- ссылки на изображения (с корректным basket-host)

На выходе формируются два файла:
- `catalog_full.xlsx` — полный набор товаров
- `catalog_filtered.xlsx` — товары, прошедшие фильтр (логика в `app/pipeline/filters.py`)

---

## Требования

- Python 3.11+ (желательно 3.12/3.13)
- `uv` (рекомендуется) или любой способ запуска Python-проекта
- Доступ в интернет
- Куки Wildberries (для доступа к данным карточки, не всегда необходимо, но рекомендуется)

---

## Запуск
1. Клонируйте репозиторий и перейдите в папку проекта, затем запустите парсер с нужными параметрами:
   ```bash
    git clone https://github.com/maniakalochka/wb_parser
    cd wb_parser
   
    export WB_COOKIE='*укажите ваши куки*'
    uv run app --limit 30 --max-pages 1 --concurrency 1 --out-dir artifacts 
    ```
   
2. Результаты будут сохранены в папке `artifacts`:
3. - `catalog_full.xlsx` — полный набор товаров
   - `catalog_filtered.xlsx` — товары, прошедшие фильтр (логика в `app/pipeline/filters.py`)
## P.S.
   - --limit 30 — ограничение на количество товаров 
   - --max-pages 1 — ограничение на количество страниц поиска
   - --concurrency 1 — количество параллельных запросов (можно увеличить, но будьте осторожны с блокировками)
   - --out-dir artifacts — папка для сохранения результатов
   - WB_COOKIE — ваши куки для доступа к данным карточки (не всегда обязательны, но рекомендуется для получения полной информации)
